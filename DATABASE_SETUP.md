# 📊 Инструкции по загрузке базы данных

## 🎯 Выбор базы данных

Выберите одну из двух баз данных в зависимости от вашего хостинга:

### ✅ Для MySQL/MariaDB (рекомендуется)
- **Файл:** `database.sql`
- **Размер:** ~50 KB
- **Совместимость:** MySQL 5.7+, MariaDB 10.2+
- **Поддержка:** Лучше всего поддерживается на хостингах

### ✅ Для PostgreSQL
- **Файл:** `database_pg.sql`
- **Размер:** ~50 KB
- **Совместимость:** PostgreSQL 10+
- **Поддержка:** Если ваш хостинг использует PostgreSQL

## 📥 Способ 1: Через phpMyAdmin (для MySQL)

### Шаг 1: Откройте phpMyAdmin
```
http://ваш-хостинг.com/phpmyadmin
```

### Шаг 2: Авторизуйтесь
- Введите логин и пароль от БД

### Шаг 3: Создайте новую БД (опционально)
- Нажмите "Новая БД"
- Введите имя: `os_journal`
- Нажмите "Создать"

### Шаг 4: Импортируйте SQL файл
1. Откройте вкладку "Импорт"
2. Нажмите "Выбрать файл"
3. Выберите `database.sql`
4. Нажмите "Выполнить"

### Шаг 5: Проверьте результат
- Должны появиться таблицы в БД
- Должны появиться тестовые данные

## 📥 Способ 2: Через SSH (для Linux хостинга)

### Для MySQL:
```bash
# Подключитесь к серверу по SSH
ssh user@your-host.com

# Перейдите в директорию проекта
cd public_html

# Загрузите БД
mysql -u root -p os_journal < database.sql

# Введите пароль от БД
```

### Для PostgreSQL:
```bash
# Подключитесь к серверу по SSH
ssh user@your-host.com

# Перейдите в директорию проекта
cd public_html

# Загрузите БД
psql -U postgres -d os_journal -f database_pg.sql

# Введите пароль от БД
```

## 📥 Способ 3: Через FTP + phpMyAdmin

2. Подключитесь к серверу
3. Загрузите `database.sql` в корневую директорию

1. Откройте phpMyAdmin

3. Откройте вкладку "Импорт"

5. Нажмите "Выполнить"

## 🔧 Конфигурация подключения

### Отредактируйте `.env.local`:
```bash

DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASS=ваш_пароль
DB_NAME=os_journal

```

### Или отредактируйте `db.php`:
```php

define('DB_PORT', '3306');
define('DB_USER', 'root');
define('DB_PASS', 'ваш_пароль');
```

## 📊 Содержимое БД

### Таблицы:
- `regions` - 8 районов Алматы
- `users` - 3 пользователя (admin, moderator, viewer)
- `commissions` - 6 комиссий
- `os_members` - 10 членов ОС
- `incoming_letters` - 5 входящих писем
- `outgoing_letters` - 3 исходящих письма
- `letter_members` - связь писем с членами
- `letter_recipients` - получатели писем
- `letter_scans` - сканы писем
- `audit_logs` - логи аудита
- `user_sessions` - сессии пользователей

### Пользователи для входа:
```
Логин: admin
Пароль: password
Роль: Администратор

Логин: moderator
Пароль: password
Роль: Модератор

Логин: viewer
Пароль: password
Роль: Просмотрщик
```

## ⚠️ Решение проблем

### Ошибка: "Access denied for user"
- Проверьте логин и пароль БД
- Проверьте, что пользователь имеет права на БД

### Ошибка: "Database does not exist"
- Создайте БД `os_journal` вручную
- Или используйте SQL файл, который создает БД

### Ошибка: "Table already exists"
- Удалите старую БД
- Или используйте другое имя БД

### Ошибка: "Syntax error"
- Проверьте версию MySQL/PostgreSQL
- Убедитесь, что используете правильный SQL файл
