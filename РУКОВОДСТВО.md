## 1. Что такое система и для кого она

**Журнал ОС** — веб-система для ведения делопроизводства Общественного Совета города Алматы. Система автоматизирует:

- регистрацию входящей и исходящей корреспонденции;
- отслеживание сроков ответа (15 рабочих дней по умолчанию);
- назначение ответственных членов ОС;
- формирование ежемесячных отчётов;
- контроль исполнительской дисциплины (KPI).

**Целевая аудитория:** секретари, члены Общественного Совета, администраторы системы.

## 2. Технологический стек

| Слой | Технология |
|------|-----------|
| Бэкенд | PHP 8.0+ (без фреймворка, PSR-4 автозагрузка) |
| База данных | MySQL 5.7+ / 8.0 (SQLite для тестов) |
| Фронтенд | Vanilla JS SPA, Bootstrap 5.3, Bootstrap Icons 1.13 |
| Real-time | Pusher Channels (WebSocket) |
| Email | PHPMailer 6.x + очередь в БД |
| Telegram | Bot API (webhook) |
| Авторизация | PHP-сессии + TOTP 2FA (Google Authenticator) |
| Безопасность | CSRF-токены с ротацией, Rate Limiter (файл/Redis), SRI |
| Тестирование | PHPUnit 10.5 |
| Перевод | KazLLM (Ollama/llama.cpp) — опционально |
| Контейнеры | Docker Compose (опционально) |

## 4. Роли пользователей и права доступа

В системе три роли. В базе данных хранятся значения `admin`, `manager`, `viewer`; в PHP-коде роль `manager` нормализуется до `moderator`.

| Роль | В БД | Что может делать |
|------|------|-----------------|
| **Администратор** | `admin` | Полный доступ ко всем функциям, всем регионам, управление пользователями, просмотр аудит-лога, экспорт, импорт. Обязательно должен настроить 2FA перед первым входом. |
| **Модератор** | `manager` | Работа с письмами своего региона: создание, редактирование, добавление сканов, назначение членов. Экспорт ограничен. Может переключать регион (если назначен). |
| **Наблюдатель** | `viewer` | Только просмотр данных своего региона. Не может создавать или изменять записи. |

### Детальная матрица прав

| Действие | admin | moderator | viewer |
|----------|-------|-----------|--------|
| Просмотр писем | ✅ все регионы | ✅ свой регион | ✅ свой регион |
| Создание/редактирование писем | ✅ | ✅ | ❌ |
| Удаление (архивирование) писем | ✅ | ✅ | ❌ |
| Пакетное архивирование | ✅ | ✅ | ❌ |
| Восстановление из архива | ✅ | ✅ | ❌ |
| Загрузка сканов | ✅ | ✅ | ❌ |
| Управление членами ОС | ✅ | ✅ | ❌ |
| Управление комиссиями | ✅ | ✅ | ❌ |
| Управление мероприятиями | ✅ | ✅ | ❌ |
| Экспорт JSON/CSV/XLSX | ✅ | ограничен | ❌ |
| Экспорт PDF | ✅ | ✅ | ❌ |
| Импорт CSV | ✅ | ❌ | ❌ |
| Управление пользователями | ✅ | ❌ | ❌ |
| Управление регионами | ✅ | ❌ | ❌ |
| Просмотр аудит-лога | ✅ | ❌ | ❌ |
| Панель администратора | ✅ | ❌ | ❌ |
| Переключение активного региона | ✅ | ✅ (если есть) | ❌ |

## 5. Функциональные модули

### 5.1 Входящие письма

**Раздел «Входящие»** — журнал входящей корреспонденции.

**Поля письма:**
- Регистрационный номер (Вх.N) — автоматически
- Дата входящего
- Организация-отправитель
- Категория (государственный орган / житель / организация)
- Номер ҚК (Казахстанский классификатор)
- Тема / краткое содержание
- Адресаты — выбор из реестра членов ОС
- Ответственные исполнители — выбор из членов ОС
- Дедлайн (авторасчёт: 15 рабочих дней)
- Прикреплённые сканы (PDF, DOCX, JPG, PNG, MP4 и др.)
- Примечание
- Связанное исходящее письмо (ответ)

**Фильтры:**
- По году и месяцу
- По статусу (все / ожидает ответа / просрочено)
- По наличию сканов (со сканами / без сканов)
- По адресату

**Функции:**
- Добавление письма через форму (сворачиваемую)
- Редактирование существующего письма
- Просмотр детальной карточки (вкладки: Основное, Комментарии, История)
- Загрузка нескольких сканов одновременно
- Пакетный выбор (checkbox) и экспорт / архивирование выбранных
- Цветовая индикация дедлайна: зелёный / жёлтый / красный
- Пагинация (25 писем на страницу по умолчанию)
- Real-time обновление при изменениях другими пользователями (Pusher)

---

### 5.2 Исходящие письма

**Раздел «Исходящие»** — журнал ответов и самостоятельных писем ОС.

**Поля письма:**
- Порядковый номер (Исх.N)
- Дата отправки
- Исходящий номер (вручную или авто)
- Тип адресата (ҚК / ЖТ / ЗТ)
- Организация-получатель
- Привязка к входящему письму (если это ответ)
- Тема / содержание
- Прикреплённые файлы
- Примечание

**Функции:**
- При создании можно выбрать «Связать с входящим» — тогда входящее помечается как «отвечено» и убирается из просроченных
- Пакетный выбор и архивирование
- Все фильтры аналогичны входящим

---

### 5.3 Архив

**Раздел «Архив»** — мягкое удаление. Письма не удаляются из БД, а помечаются `deleted_at`.

- Просмотр архивированных входящих и исходящих (отдельные вкладки)
- Восстановление по одному (кнопка «Восстановить»)
- Пакетное архивирование из основного журнала (до 100 писем за раз)

---

### 5.4 Комментарии и история

В детальной карточке каждого письма доступны две вкладки:

**Комментарии:**
- Добавление текстового комментария
- Просмотр всех комментариев с датой и автором
- Изменение и удаление своих комментариев (только автор / admin)
- Real-time обновление

**История изменений:**
- Журнал всех действий с письмом: кто, когда, что изменил
- Показывается старое и новое значение поля

---

### 5.5 Шаблоны писем

- Сохранение часто используемых формулировок как шаблон (название + текст)
- Вставка шаблона в поле «Тема» одним кликом при создании письма
- Управление шаблонами (создание, редактирование, удаление)
- Шаблоны общие для всего региона

---

### 5.6 Члены Общественного Совета

**Раздел «Члены ОС»** — реестр членов для назначения в письма.

**Поля члена:**
- ФИО (полное)
- Должность
- Комиссия (привязка)
- Фотография (загрузка, хранение в `uploads/photos/`)
- Статус активности
- Контакты

**Функции:**
- Добавление, редактирование, удаление члена
- Загрузка фотографии (поддерживаются JPEG, PNG; проверка реального MIME через `finfo`)
- Статистика по члену: сколько писем назначено, сколько отвечено, KPI
- Привязка члена к комиссии

---

### 5.7 Комиссии

- Создание и управление комиссиями ОС
- Привязка членов к комиссиям
- Фильтрация членов по комиссии

---

### 5.8 Мероприятия и явка

**Раздел «Мероприятия»** — учёт заседаний и событий.

**Поля мероприятия:**
- Название
- Дата и время
- Место проведения
- Тип (заседание / выезд / встреча и др.)
- Описание

**Функции:**
- Отметка явки членов ОС на мероприятие
- KPI явки: процент посещаемости по каждому члену
- Фильтрация мероприятий по периоду

---

### 5.9 Дашборд и KPI

**Главный экран** после входа содержит сводную панель.

**Виджеты KPI:**
- Всего входящих / без ответа / просроченных
- Всего исходящих
- Мероприятий в этом месяце
- Активных членов ОС

**Графики (Chart.js):**
- Динамика писем по месяцам (входящие vs исходящие)
- Распределение по категориям
- Топ исполнителей по количеству ответов
- Явка членов на мероприятия

**Расширенная статистика (admin):**
- Сравнение регионов по объёму писем
- Тепловая карта активности по дням
- Средний срок ответа

---

### 5.10 Календарь дедлайнов

- Отображение входящих писем без ответа в виде календаря
- Цветовая индикация по сроку: зелёный / жёлтый (≤3 дней) / красный (просрочено)
- Навигация по месяцам
- Клик на дату — список писем за этот день

---

### 5.11 Глобальный поиск

- Поиск по номеру, организации, теме, содержанию — одновременно по входящим и исходящим
- Подсветка найденного текста в результатах
- Результаты разбиты по типу письма
- Доступен из любого раздела (иконка поиска в боковом меню)

---

### 5.12 Экспорт данных

| Формат | Содержимое | Кто может |
|--------|-----------|-----------|
| **JSON** | Все поля писем, вложенные данные | admin |
| **CSV** | Плоская таблица писем | admin |
| **XLSX** | Excel-файл с двумя листами (входящие + исходящие) | admin |
| **PDF** | Отчётный документ | admin, moderator |
| **Пакетный JSON** | Только выбранные письма | все, у кого есть доступ |

Каждый экспорт фиксируется в аудит-логе (кто, когда, сколько записей).

---

### 5.13 Импорт писем

- Загрузка CSV-файла с письмами
- Автоматическое определение кодировки
- Валидация полей перед импортом
- Отчёт об ошибках по строкам
- Только для роли `admin`

---

### 5.14 Уведомления

Уведомления поступают тремя каналами:

**1. В браузере (Web Push / Pusher):**
- Real-time оповещение при новых письмах и изменениях
- Значок непрочитанных в боковом меню
- Центр уведомлений (список с пометкой «прочитано»)

**2. Email:**
- Уведомление о приближающемся дедлайне (за 3 дня и в день срока)
- Ежемесячный отчёт с таблицей писем за месяц
- Уведомление о назначении исполнителем
- Все email отправляются через очередь (`email_queue` в БД)

**3. Telegram:**
- Уведомление при назначении члена ОС ответственным за письмо
- Команды бота (см. раздел 6)

---

### 5.15 Профиль пользователя

Каждый пользователь может управлять своим аккаунтом:

- **Изменить отображаемое имя** (ФИО)
- **Изменить email**
- **Изменить пароль** (требуется текущий пароль)
- **Загрузить/удалить фото профиля** (JPEG, PNG, макс. 5 МБ; реальный MIME проверяется через `finfo`)
- **Настроить 2FA** (TOTP: Google Authenticator, Яндекс.Ключ, Authy)
- **Привязать Telegram** (получение 6-значного кода → отправить боту `/link <код>`)
- **Сброс пароля** (через email — ссылка действительна 1 час)

### 5.16 Аудит-лог

Все изменения в системе записываются в таблицу `audit_logs`.

Фиксируются события:
- Вход / выход из системы
- Создание / изменение / удаление любой записи
- Попытки неудачного входа (`LOGIN_FAILED`)
- Нарушения CSRF (`CSRF_VIOLATION`)
- Превышение лимита запросов (`RATE_LIMIT`)
- Скачивание сканов, экспорт данных

**Просмотр** доступен только администратору (`/api/audit_logs.php`).

**Очистка:** автоматически через `cron_cleanup.php` (настраивается переменной `AUDIT_LOG_RETENTION_DAYS`, по умолчанию 365 дней).

## 8. Безопасность

### Аутентификация

- Вход по логину и паролю (bcrypt хеш)
- **2FA (TOTP):** обязательна для роли `admin`; опциональна для остальных
  - Совместима с Google Authenticator, Яндекс.Ключом, Authy
  - Настройка через профиль: сканирование QR-кода → ввод 6-значного кода для подтверждения
- **Сброс пароля:** по email, токен действует 1 час, одноразовый
- **Ограничение попыток входа:** 10 попыток за 15 минут с одного IP

### CSRF-защита

- Каждый POST/PUT/DELETE запрос требует заголовок `X-CSRF-Token`
- Токен ротируется после каждого успешного мутирующего запроса
- Новый токен возвращается в заголовке `X-New-CSRF-Token`
- JS-перехватчик (`csrf-handler.js`) автоматически добавляет заголовок и обновляет кэш

### Rate Limiting (ограничение запросов)

- По умолчанию: 100 запросов в час на пользователя/IP
- Сброс пароля: 5 попыток за 15 минут с IP
- Telegram webhook: 30 сообщений в минуту на chat_id
- Хранение: в файлах (`.rate_limit/`) или Redis (если настроен `REDIS_HOST`)
- При превышении возвращается HTTP 429

### Redis (опционально)

Если настроены переменные окружения, Rate Limiter использует Redis вместо файловой системы — это корректно работает в кластерном окружении:

```env
REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_PASSWORD=
```

### Загрузка файлов

- Реальный MIME-тип определяется через `finfo_open(FILEINFO_MIME_TYPE)` — клиентский заголовок игнорируется
- Максимальный размер: 5 МБ для фото, 20 МБ для сканов (настраивается)
- Прямой доступ к `uploads/` закрыт через `.htaccess` — файлы отдаются только через API с проверкой прав
- Защита от path traversal при сохранении файлов

### Аудит безопасности

События безопасности записываются через `SecurityAuditService`:
- Неудачный вход → `LOGIN_FAILED`
- Нарушение CSRF → `CSRF_VIOLATION`
- Превышение Rate Limit → `RATE_LIMIT`

## 12. Cron-задания

### Настройка (crontab)

```bash
# Уведомления о дедлайнах — каждый час
0 * * * * php /путь/к/проекту/cron_deadlines.php >> /var/log/os-cron.log 2>&1

# Отправка очереди email — каждые 5 минут
*/5 * * * * php /путь/к/проекту/cron_send_emails.php >> /var/log/os-email.log 2>&1

# Ежемесячный отчёт — 1-го числа в 09:00
0 9 1 * * php /путь/к/проекту/cron_monthly_report.php >> /var/log/os-monthly.log 2>&1

# Очистка устаревших данных — каждый день в 03:00
0 3 * * * php /путь/к/проекту/cron_cleanup.php >> /var/log/os-cleanup.log 2>&1
```

### Описание cron-скриптов

| Скрипт | Что делает |
|--------|-----------|
| `cron_deadlines.php` | Ищет письма с дедлайном через 3 дня и сегодня → ставит email в очередь + Telegram-уведомление |
| `cron_send_emails.php` | Берёт до 10 писем из `email_queue` со статусом `queued` → отправляет через SMTP |
| `cron_monthly_report.php` | Формирует HTML-отчёт за прошлый месяц по каждому региону → отправляет admin и moderator |
| `cron_cleanup.php` | Удаляет: старые аудит-логи, просроченные токены (telegram/password), отработанные email, файлы rate limiter |

### Вызов через HTTP (альтернатива cron)

```
GET /api/cron_deadlines.php?token=<CRON_TOKEN>
```

## 15. Интернационализация

Интерфейс поддерживает **русский** и **казахский** языки.

- Переключатель языка в шапке приложения
- Файлы переводов: `api/js/i18n.js` (словари), `api/js/i18n-dom.js` (применение к DOM)
- Выбор языка сохраняется в `localStorage`
- Email-уведомления отправляются на русском

**KazLLM-перевод (опционально):**
Если в `.env` включён `KAZLLM_ENABLED=true` и настроен локальный Ollama-сервер, система может автоматически переводить пользовательский контент (темы писем, примечания) на казахский через `/api/translate.php`.

## 16. PWA и мобильный вид

### Progressive Web App

Приложение устанавливается как PWA:
- `api/sw.js` — Service Worker, кэширует статические ресурсы
- `api/manifest.json` — описание приложения (иконка, название, цвет)
- На iOS/Android: «Добавить на экран» → работает офлайн (кэшированные страницы)

### Мобильный интерфейс

- Нижняя навигационная панель на экранах < 768px
- Таблицы писем преобразуются в карточки (CSS data-label)
- Свайп-навигация между разделами
- Touch-оптимизированные элементы управления

### Тёмная тема

Переключение через иконку в шапке. Предпочтение сохраняется в `localStorage`. Поддерживает системную тему (`prefers-color-scheme`).
